# A. 粘合剂数据线

## 目标

把当前空白讨论稿收敛为一个针对明确粘合剂体系、明确性能目标和统一测试条件的 v1.0 数据接口，并取得首批可审计的真实数据。

当前模板：[`data_templates/adhesive/粘合剂重要化学性质_数据格式_v0.3.xlsx`](../../data_templates/adhesive/%E7%B2%98%E5%90%88%E5%89%82%E9%87%8D%E8%A6%81%E5%8C%96%E5%AD%A6%E6%80%A7%E8%B4%A8_%E6%95%B0%E6%8D%AE%E6%A0%BC%E5%BC%8F_v0.3.xlsx)。它是字段讨论稿，不是现成数据集。

## 化学组需要确认和填写什么

| 类别 | 需要确认的内容 | 为什么需要 |
|---|---|---|
| 研究边界 | 粘合剂化学体系、样品/配方的定义 | 防止把不可比体系混在一起 |
| 预测目标 | 一个主要性能、单位、测试标准 | 保证标签含义一致 |
| 配方信息 | 组分身份、用量或比例、批次 | 构成模型主要输入并支持追溯 |
| 关键理化性质 | 实际测得且对当前体系可获得的性质 | 形成表格模型输入，不要求为了填表而虚构测量 |
| 工艺条件 | 混合、固化、温湿度、时间 | 性能通常同时受配方和工艺影响 |
| 测试条件 | 基材、表面处理、胶层厚度、加载方式 | 判断不同标签值能否直接比较 |
| 质量信息 | 重复次数、误差、缺失原因、来源 | 便于数据审计和不确定性分析 |

具体必填与条件填写规则以工作簿说明页和浅蓝色单元格为准。无法获得的字段应注明“未测量/未知”，不能估一个数值填入。

## 交付顺序

1. **字段确认**：导师与化学组删除无关列、补充本体系必须列，并统一单位。
2. **格式验收**：填写 3–5 行真实样例；这里只验证含义和读取流程。
3. **首批数据**：按确认后的 v1.0 接口提供真实记录，同时保留样品编号与来源。
4. **数据审计**：检查重复、单位、缺失、异常值、批次和潜在数据泄漏。
5. **冻结版本**：保存不可覆盖的原始版，并生成供建模使用的处理版。

## 完成标准

- 每行代表什么已经唯一确定；
- 预测目标、单位和测试方法唯一确定；
- 重要实验条件能够追溯；
- 缺失值与重复实验有明确记录；
- 数据共享权限已经确认；
- 算法组可以无人工猜测地读取每一列。

## 不属于这条线的工作

- 不在真实数据到达前选择“最终最佳模型”；
- 不把不同论文、不同材料体系和不同测试方法的数据直接拼表；
- 不因为算法需要某一列就编造实验值；
- 不在未授权时把合作组原始数据上传 GitHub。
