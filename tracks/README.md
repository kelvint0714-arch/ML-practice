## 为什么拆成两条线

粘合剂体系、预测目标和测试条件仍需导师与化学组确认，暂时不能直接训练项目模型。等待期间，算法侧可以先在公开数据上建立可复现流程。真实数据到达后，应复用同一评估框架，而不是从头重写。

| 工作线 | 要回答的问题 | 当前材料 | 不能越过的边界 |
|---|---|---|---|
| [A. 粘合剂数据线](01_data/README.md) | 项目实际收集什么、每一列怎样定义？ | 粘合剂空白模板 v0.3 | 不自行编造或用不同论文数值拼成“真实数据” |
| [B. 算法预研线](02_algorithm/README.md) | 小样本表格数据上什么模型与验证方式可信？ | ESOL Day 1、公开论文数据 | 不把公开数据结果包装成本项目粘合剂结论 |

## 两条线如何衔接

```text
数据线：体系/目标/条件确认 → 3–5 行样例验收 → 首批真实数据 → 数据审计
                                         ↓
算法线：传统基线 → MLP 对照 → 混合模型 → 不确定性/主动学习 → 接入真实数据
```

3–5 行样例只用于检查字段、单位和读取代码，不能用于训练或比较模型。接入真实数据前，需要确认重复实验、批次、配方分组和测试条件，之后再决定交叉验证或留出策略。

## 汇合后才可以做的事

- 在真实粘合剂数据上比较传统模型、MLP 和混合模型；
- 根据实际样本量决定模型复杂度；
- 如果获得 SMILES、分子图或结构文件，再增加分子描述符或 GNN；
- 在导师确认后，将模型建议用于下一轮实验设计。

## GitHub 数据边界

- 空白模板、字段说明和公开来源可以进入仓库；
- 真实实验数据上传前必须确认保密、署名和共享权限；
- 未确认前只提交字段说明、脱敏样例或读取脚本，不上传原始实验记录。
